# SyncGhost Vanilla SPA - 快速启动指南

## 🚀 立即开始

### 方式1：使用 Python（推荐）

```bash
cd vanilla-spa
python -m http.server 8000
```

然后访问：http://localhost:8000

### 方式2：使用 Node.js

```bash
cd vanilla-spa
npx http-server -p 8000
```

然后访问：http://localhost:8000

### 方式3：使用 PHP

```bash
cd vanilla-spa
php -S localhost:8000
```

然后访问：http://localhost:8000

## ⚠️ 重要提示

**不能直接双击打开 index.html！**

由于使用了 ES6 模块，必须通过 HTTP 服务器运行。直接打开 HTML 文件会遇到 CORS 错误。

## 📖 功能演示流程

### 1. 首次进入（Onboarding）

首次打开应用会自动进入欢迎页：

1. 点击"开始使用"按钮
2. 弹出"添加网盘账号"对话框
3. 输入账号别名（例如：我的备份）
4. 选择网盘类型（百度网盘/一刻相册）
5. 如果选择百度网盘或一刻相册，会显示二维码
6. 点击"确认绑定"
7. 弹出"授权成功"对话框
8. 点击"立即配置任务"
9. 弹出"添加同步任务"对话框
10. 填写任务信息：
    - 本地路径：D:\Photos
    - 网盘账号：选择刚才添加的账号
    - 远程路径：/备份/照片
    - 同步方向：选择"备份"
11. （可选）点击"高级设置"展开更多选项
12. 点击"创建任务"
13. 自动跳转到仪表盘
14. 显示绿色 Toast："配置完成，开始同步"

### 2. 仪表盘功能

- 查看实时上传/下载速度（每2秒更新）
- 查看队列概览和运行状态
- 监控实时传输列表
  - 正在上传的任务（进度条动态更新）
  - 正在扫描的任务（无限滚动进度条）
  - 已完成的任务（灰色显示）
- 查看终端日志
  - 日志自动滚动
  - 点击"清空日志"清空
  - 日志会每5秒自动添加
- 点击"初次进入"可以重新进入引导流程
- 点击"立即同步"触发手动同步

### 3. 同步任务功能

- 查看所有任务列表
- 点击"添加任务"创建新任务
- 点击任务的"编辑"按钮修改任务
- 点击"暂停/启动"按钮切换任务状态
- 点击"删除"按钮删除任务（需确认）

### 4. 网盘账号功能

- 查看所有已绑定的账号
- 点击"添加账号"或虚线卡片添加新账号
- 点击"解绑账号"解绑账号（需确认）
- 绑定成功后可选择立即配置任务

### 5. 系统设置功能

- 修改 Web 服务配置
- 修改日志配置
- 修改性能配置
- 点击"保存设置"保存
- 点击"重置为默认"恢复默认值（需确认）

## 🎯 测试性能优化

### 测试日志性能

1. 打开浏览器开发者工具（F12）
2. 切换到 Performance 标签
3. 开始录制
4. 等待日志自动添加（每5秒一条）
5. 或者手动修改 `Dashboard.js` 中的间隔时间，改为100ms
6. 观察 CPU 占用和 FPS
7. 你会发现即使添加大量日志，性能依然流畅
8. 这得益于增量渲染和内存控制优化

### 测试数据更新节流

1. 打开开发者工具
2. 观察上传/下载速度的更新频率
3. 即使后端推送频率很高，UI 更新最多每0.5秒一次
4. 避免了高频更新导致的卡顿

## 💡 常见问题

### Q: 为什么不能直接打开 HTML？
A: 因为使用了 ES6 模块，浏览器的 CORS 策略要求必须通过 HTTP(S) 协议访问。

### Q: 如何清除 Onboarding 状态？
A: 打开浏览器控制台，执行：
```javascript
localStorage.removeItem('syncghost_onboarded');
location.reload();
```

### Q: 如何修改初始数据？
A: 编辑对应页面的 constructor 中的初始数据数组。

### Q: 如何添加新功能？
A: 参考 README.md 中的"扩展开发"章节。
