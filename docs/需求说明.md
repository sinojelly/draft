写一个简单的代码备份工具，运行它，传入一个源目录路径，一个规则文件路径，一个目的目录路径，就会把源目录符合规则的文件，保持目录结构拷贝到目的目录路径。可以用参数控制，是否先清除目的目录所有文件。默认是清除的。文件拷贝完，在目的目录 执行 git add . ; git commit -m "auto commit" ; git push 。



名字就用 RuleSync 吧。

规则可以复用 .gitignore， 支持指定哪些文件、哪些文件夹要拷贝即可。

-rules 没指定，就使用 源路径下的 .rulesync 文件。



为了提升安全性，在 rulesync.exe 工具所在目录，自动写一个文件记录历史上使用的 dest 路径。

如果传入的 dest 路径不在历史路径中，则询问用户是否要用它作为历史路径，将会清空该路径所有文件。



软件运行中，最好给出反馈，比如拷贝了哪些文件，git 命令执行情况等。
